# Ont-Sim

Система оценки семантической близости предметных областей.

## Запуск

```bash
docker compose up
```

Swagger UI:

```text
http://localhost:8000/docs
```

Проверка работоспособности:

```text
http://localhost:8000/health
```

---

## Остановка

```bash
docker compose down
```

---

## Особенности запуска на macOS (Apple Silicon)

Для локального запуска был изменён `docker-compose.yml`, чтобы проект работал в CPU-режиме на macOS.

Изменение сохранено в локальном коммите:

```text
Fix Docker configuration for CPU mode on macOS
```

---

## Текущее состояние проекта

Проект запускается через Docker.

Работают:

* Swagger API
* загрузка терминов через `/api/v1/upload/json`
* получение списка доменов `/api/v1/domains`

---

## Ограничения публичной версии

В публичном репозитории отсутствуют:

* исходный корпус терминов;
* пайплайн автоматического сбора данных из Wikipedia и arXiv;
* скрипты построения датасетов;
* готовые benchmark-датасеты;
* построенные RAG-индексы.

Проверка RAG:

```text
GET /api/v1/rag/status
```

возвращает:

```json
{
  "built": false
}
```

---

## Что удалось выяснить из ВКР

Согласно описанию дипломной работы, данные должны формироваться автоматически:

1. Сбор текстов из Wikipedia через MediaWiki API.
2. Сбор аннотаций из arXiv через OAI-PMH.
3. Очистка текста.
4. Токенизация.
5. Построение униграмм, биграмм и триграмм.
6. Лемматизация.
7. Фильтрация стоп-слов.
8. Валидация через POS-теги и RuWordNet.
9. Расчёт TF-IDF.
10. Формирование терминов по доменам.

Однако соответствующий код отсутствует в публичной версии репозитория.

## Данные

На текущий момент данные необходимо загружать вручную через API.

Пример домена:

```json
{
  "domains": [
    {
      "name": "ML",
      "terms": [
        {
          "name": "машинное обучение",
          "frequency": 10
        }
      ]
    }
  ]
}
```
